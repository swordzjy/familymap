# 首页引导优化 PRD

**版本：** 1.2  
**日期：** 2026-04-09  
**目标：** 提升中老年用户聊天引导接受度，降低冷启动流失率
**状态：** ✅ 已完成

---

## 背景

### 产品定位
"寻根"是一款通过自然语言聊天帮助用户完成家族数据输入的应用，自动解析并结构化家族血缘关系、人物信息、事件（时间地点人物事）等数据。

### 当前问题
- 新用户首次进入为传统 AI 聊天窗口
- 开场白平淡："先说说你自己吧，你现在住在哪里"
- 中老年用户不知道如何开始聊天
- 用户聊几句就流失，缺乏回访机制

### 用户画像
- 有阅历的中老年人（50-70 岁）
- 成功者、经历复杂丰富者居多
- 可能不擅长打字，偏好语音输入
- 需要温暖、尊重的交互体验

### 2. 断点续聊机制（优先级：高）✅ 已完成

**目标：** 降低中途流失，提升回访完成度

**数据存储：**
- 临时状态：localStorage（用户清理缓存会丢失）
- 关键数据：实时同步到 DB（家族关系、迁徙事件等）

**UI 行为：**
1. 用户返回聊天页面时，检查 localStorage 是否有未完成对话
2. 如果有，显示"继续上次的对话"横幅
3. 点击后 AI 主动追问上次的话题

**数据存储结构：**
```javascript
localStorage.setItem('chatSession', JSON.stringify({
  familyId: "xxx",
  lastPrompt: "您说在黑龙江待过，那是哪一年？",
  lastUserInput: "大概是 1985 年左右吧",
  messages: [...],  // 完整对话历史
  lastActiveTime: "2026-04-09T10:30:00Z"
}));
```

**追问话术示例：**
- "欢迎回来！上次您说到 1985 年左右在黑龙江，后来呢？"
- "您之前提到因为工作搬到了南方，能具体说说是哪一年吗？"
- "我们刚记录了您的出生地，接下来您离开家乡后去了哪里？"

**验收标准：**
- [x] 刷新/返回后显示"继续对话"提示
- [x] 点击后恢复历史对话
- [x] 无未完成对话时不显示提示
- [x] 关键数据已实时入库，localStorage 仅用于对话上下文

**实现说明：**
- 前端：`index.html:675-677` - 断点续聊横幅 HTML
- 前端：`index.html:347-389` - 横幅 CSS 样式
- 前端：`index.html:1500-1580` - JS 函数（checkResumeChat, resumeChat, saveChatSession）
- 移动端：`mobile-chat.html:320-326` - 横幅 HTML
- 移动端：`mobile-chat.html:257-293` - 横幅 CSS 样式
- 移动端：`mobile-chat.html:793-872` - JS 函数

### 3. 示例对话气泡（优先级：中）✅ 已完成

**目标：** 解决"不知道说什么"的冷启动问题

**UI 设计：**
- 弹幕式横向滚动，节省垂直空间
- 点击只填充输入框，不自动发送
- 用户可修改后手动发送

**样式变更：**
```css
.guidance-cards {
  display: flex;
  overflow-x: auto;
  -webkit-overflow-scrolling: touch;
  scrollbar-width: none;
}
.guidance-card {
  display: inline-flex;
  flex-shrink: 0;
  max-width: 280px;
  margin: 0 4px;
}
```

**交互逻辑：**
- 点击气泡 → 填充输入框 + 聚焦
- 不自动发送，用户可修改后点击发送
- 气泡文案简洁，提示语缩短

**验收标准：**
- [x] 横向滚动，节省空间
- [x] 点击只填充，不自动发送
- [x] PC 端和移动端样式适配

**实现说明：**
- 移动端：`mobile-chat.html:83-110` - 弹幕式 CSS
- 移动端：`mobile-chat.html:464-468` - useGuidance 函数修改
- 移动端：`mobile-chat.html:41-54` - messages-wrapper 可快速滑动
- PC 端：已有 suggestion-chips wrap 布局，useGuidance 已只填充不发送
