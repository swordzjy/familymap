**原则**：AI 分析结果缓存（Analysis Result Caching）

**核心思想**：
> 用户的输入 → AI 分析 → 结构化结果 → 持久化存储
> 
> 后续访问直接使用存储结果，跳过 AI 分析

**适用场景**：
- 表单/对话内容提取结构化数据
- 文档分析、摘要生成
- 代码分析、重构建议
- 任何"输入固定 → 输出可复用"的 AI 场景

**实现模式**：

```
┌─────────────────────────────────────────────────────┐
│  用户触发动作（如"整理数据"）                          │
└─────────────────┬───────────────────────────────────┘
                  │
         ┌────────▼────────┐
         │ 有已存储的结果 ID？│
         └────┬─────┬──────┘
              │是   │否
              ▼     ▼
     ┌────────────┐ ┌──────────────┐
     │ 直接加载数据 │ │ 调用 AI 分析   │
     │ <1 秒       │ │ 2-5 秒        │
     └────────────┘ └──────┬───────┘
                           │
                           ▼
                    ┌──────────────┐
                    │ 持久化结果 ID │
                    │ 返回结构化数据 │
                    └──────────────┘
```

**关键设计点**：

1. **结果 ID 绑定**：分析完成后保存唯一 ID（如 familyId），后续通过 ID 加载
2. **加载优先**：触发动作时先检查是否有 ID，有则直接加载
3. **重新分析入口**：提供"重新分析"按钮，用于修正 AI 错误
4. **状态提示**：按钮文字区分"解析中"和"加载中"

**代码模板**：

```javascript
// 主入口：智能选择加载或分析
function handleGenerate() {
  if (cachedResultId) {
    loadExistingResult(cachedResultId);  // 快速路径
  } else {
    callAnalysisAPI();  // 首次分析
  }
}

// 加载已有结果
async function loadExistingResult(resultId) {
  const data = await fetch(`/api/result/${resultId}`);
  useData(data);
}

// 重新分析（用户主动触发）
function reanalyze() {
  if (!confirm('确定重新分析？')) return;
  clearCache();
  callAnalysisAPI();
}
```

**收益**：
- 用户体验：响应速度提升 5-10 倍
- 成本：AI 调用次数减少 90%+（用户反复查看场景）
- 体验一致性：同一输入始终看到同一输出

**如何应用**：
- 设计任何 AI 分析功能时，默认采用此模式
- 不要每次用户点击都调用 AI
- 提供"刷新/重新分析"入口作为例外通道
